# Skill: 测试策略设计

## 触发条件

收到测试任务时，先用此 skill 设计测试策略再编写用例。

## 测试金字塔

```
        ╱╲
       ╱ E2E ╲        少量端到端测试 (10%)
      ╱────────╲
     ╱ 集成测试 ╲      中等集成测试 (30%)
    ╱──────────────╲
   ╱   单元测试     ╲   大量单元测试 (60%)
  ╱──────────────────╲
```

## 用例设计方法

### 等价类划分

- 有效等价类: 正常输入范围
- 无效等价类: 超出范围、错误类型、空值

### 边界值分析

- 最小值、最小值-1、最小值+1
- 最大值、最大值-1、最大值+1
- 零值、空值、单元素

### 错误猜测

- 除以零
- 空字符串/空列表
- 极大数/极小数
- Unicode/特殊字符
- 负数（当预期正数时）

## 每个函数的最小测试集

1. 正常路径（至少 2 个用例）
2. 边界值（至少 3 个用例）
3. 异常输入（至少 2 个用例）
4. 性能基准（1 个大数据量用例）
