# Skill: Bug 追踪

## 触发条件

发现测试失败或异常行为时使用。

## Bug 生命周期

```
发现 → 记录 → 通知 Forge（代码开发）→ 修复中 → 回归测试 → 关闭
                                  ↑                    │
                                  └── 未通过 ──────────┘
```

## Bug 报告模板

```markdown
### BUG-[编号]: [简短标题]

**严重程度**: Critical / Major / Minor / Trivial
**发现日期**: YYYY-MM-DD
**状态**: Open / In Progress / Fixed / Verified / Closed

**现象**:
[观察到的错误行为]

**复现步骤**:

1. [步骤 1]
2. [步骤 2]
3. [观察到错误]

**预期行为**:
[应该发生什么]

**实际行为**:
[实际发生了什么]

**测试环境**:

- 语言/版本:
- 操作系统:

**相关代码**:
[文件:行号]

**建议修复方向**:
[如果有思路]
```

## 严重程度定义

| 级别     | 定义                     | 示例               |
| -------- | ------------------------ | ------------------ |
| Critical | 功能完全不可用或数据损坏 | 核心算法结果错误   |
| Major    | 功能部分失效，无绕过方案 | 边界条件崩溃       |
| Minor    | 功能受影响但有绕过方案   | 特定输入格式不支持 |
| Trivial  | 不影响功能               | 日志格式不一致     |
