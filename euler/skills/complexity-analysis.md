# Skill: 复杂度分析

## 触发条件

当需要分析算法或代码的时间/空间复杂度时使用。

## 分析框架

1. **识别基本操作**: 确定循环、递归、数据结构操作
2. **计数**: 用 Big-O 表示最坏情况，补充平均情况和最好情况
3. **空间分析**: 栈深度、辅助数据结构、输入输出不计
4. **递推关系**: 递归算法给出递推式并用主定理或递归树求解
5. **实际影响**: 常数因子、缓存友好性、I/O 瓶颈

## 输出格式

```markdown
## 复杂度分析: [算法名]

| 指标 | 最好 | 平均 | 最坏 |
| ---- | ---- | ---- | ---- |
| 时间 | O(?) | O(?) | O(?) |
| 空间 | O(?) | O(?) | O(?) |

### 推导过程

[关键推导步骤]

### 实际性能备注

[常数因子、缓存等实际因素]
```
