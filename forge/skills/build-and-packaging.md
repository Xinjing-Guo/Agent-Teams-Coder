# Skill: Build & Packaging

## Trigger

When setting up project build systems, packaging for distribution, or managing dependencies.

## Language-Specific Build Systems

### Python — pyproject.toml

```toml
[build-system]
requires = ["setuptools>=68.0", "wheel"]
build-backend = "setuptools.backends._legacy:_Backend"

[project]
name = "mypackage"
version = "1.0.0"
requires-python = ">=3.9"
dependencies = ["numpy>=1.24", "scipy>=1.10"]

[project.optional-dependencies]
dev = ["pytest>=7.0", "black", "mypy"]

[project.scripts]
mycli = "mypackage.cli:main"

[tool.pytest.ini_options]
testpaths = ["tests"]

[tool.black]
line-length = 88

[tool.mypy]
strict = true
```

### C/C++ — CMake

```cmake
cmake_minimum_required(VERSION 3.14)
project(mylib VERSION 1.0.0 LANGUAGES C CXX)

# Library
add_library(mylib src/core.c src/algo.cpp)
target_include_directories(mylib PUBLIC
    $<BUILD_INTERFACE:${CMAKE_CURRENT_SOURCE_DIR}/include>
    $<INSTALL_INTERFACE:include>)

# Executable
add_executable(myapp src/main.cpp)
target_link_libraries(myapp PRIVATE mylib)

# Testing
enable_testing()
add_executable(tests tests/test_core.cpp)
target_link_libraries(tests PRIVATE mylib)
add_test(NAME unit_tests COMMAND tests)

# Install
install(TARGETS mylib EXPORT mylibTargets)
install(DIRECTORY include/ DESTINATION include)
```

### Makefile (universal)

```makefile
CC = gcc
CFLAGS = -Wall -Wextra -O2
SRC = $(wildcard src/*.c)
OBJ = $(SRC:.c=.o)

all: libmylib.a

libmylib.a: $(OBJ)
	ar rcs $@ $^

%.o: %.c
	$(CC) $(CFLAGS) -Iinclude -c $< -o $@

test: all
	$(CC) $(CFLAGS) -Iinclude tests/test_main.c -L. -lmylib -o test_runner
	./test_runner

clean:
	rm -f $(OBJ) libmylib.a test_runner

.PHONY: all test clean
```

### R Package

```
mypackage/
├── DESCRIPTION        # Package metadata
├── NAMESPACE          # Exports (auto-generated by roxygen2)
├── R/                 # R source files
├── src/               # C/C++/Fortran code (optional)
├── man/               # Documentation (auto-generated)
├── tests/testthat/    # Tests
└── vignettes/         # Long-form docs
```

### Julia Package

```
MyPackage/
├── Project.toml       # Dependencies
├── src/
│   └── MyPackage.jl   # Main module
├── test/
│   └── runtests.jl    # Tests
└── docs/              # Documenter.jl
```

## Dependency Management Rules

1. Pin major versions, allow minor updates: `numpy>=1.24,<2.0`
2. Separate runtime vs dev dependencies
3. Lock files for reproducibility (requirements.txt / Cargo.lock)
4. Minimal dependencies — don't add a library for one function
