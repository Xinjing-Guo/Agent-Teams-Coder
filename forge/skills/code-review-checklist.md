# Skill: 代码自审清单

## 触发条件

代码完成后、通知 Sentinel（代码测试）前，必须走一遍此清单。

## 通用审查项

### 正确性

- [ ] 算法逻辑与 Euler（算法设计师）的方案一致
- [ ] 所有分支路径都有正确处理
- [ ] 循环终止条件正确
- [ ] 递归有正确的基准情况

### 健壮性

- [ ] 空输入/空集合已处理
- [ ] 数值边界（溢出、下溢、除零）已处理
- [ ] 类型错误已防御
- [ ] 并发安全（如适用）

### 可读性

- [ ] 函数名说明了做什么
- [ ] 变量名说明了是什么
- [ ] 复杂逻辑有注释说明为什么
- [ ] 函数长度 < 50 行（否则需要拆分）

### 性能

- [ ] 无不必要的重复计算
- [ ] 数据结构选择合理
- [ ] 大数据量下无 O(n²) 隐患（除非有理由）

### 安全

- [ ] 无命令注入风险（Shell）
- [ ] 无缓冲区溢出（C/C++）
- [ ] 用户输入已验证
