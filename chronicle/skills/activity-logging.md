# Skill: 活动日志记录

## 触发条件

团队开始新任务时启动，持续记录直到任务完成。

## 记录规则

### 必须记录的事件

- Marshall（Leader）分配任务
- Euler（算法设计师）提交算法方案
- Forge（代码开发）完成代码 / 修复 Bug
- Sentinel（代码测试）提交测试报告
- Lens（代码分析）提交分析报告
- Atlas（文档工程师）完成/更新文档
- 共享记忆变更（请求、审批、拒绝）
- 成员间的关键沟通
- 重要决策及其理由

### 可以省略的事件

- 例行通知检查（无新通知时）
- 重复的状态查询
- 未产生结论的中间讨论

## 日志文件命名

```
shared/tasks/log_[任务名]_[YYYYMMDD].md
```

## 更新总结触发条件

以下情况需要生成更新总结：

1. 任务全部完成时（Marshall（Leader）通知）
2. 工作流阶段切换时（如从"代码开发"进入"代码测试"）
3. 发生重大决策变更时
4. 用户明确要求时

## 总结分发

更新总结完成后通知：

```bash
bash ../scripts/notify.sh chronicle marshall "更新总结" "[摘要]"
bash ../scripts/notify.sh chronicle all "更新总结" "[摘要]"
```
